# 해시 실습 — 파일 무결성 / 악성코드 식별

실습자료 `hash/` 폴더. 악성코드 분석에서 **해시**는 (1) 파일이 변조됐는지 확인하고 (2) 알려진 악성코드를 식별(시그니처)하는 데 쓰인다. 동봉된 `hash.py`로 실제 파일들의 MD5/SHA3 해시를 계산해 본다.

## hash.py

```python
import hashlib

def get_md5_hash(file_path):
    hash_md5 = hashlib.md5()
    with open(file_path, "rb") as f:
        for chunk in iter(lambda: f.read(4096), b""):
            hash_md5.update(chunk)
    return hash_md5.hexdigest()

def get_sha3_hash(file_path):
    hash_sha3 = hashlib.sha3_512()
    with open(file_path, "rb") as f:
        for chunk in iter(lambda: f.read(4096), b""):
            hash_sha3.update(chunk)
    return hash_sha3.hexdigest()

file_path_1 = "./hello.exe"
file_path_2 = "./erase.exe"

print("MD5 hash:", get_md5_hash(file_path_1))
print("MD5 hash:", get_md5_hash(file_path_2))
print("/////")
print("sha3 hash:", get_sha3_hash(file_path_1))
print("sha3 hash:", get_sha3_hash(file_path_2))
```

### 코드 포인트

- `iter(lambda: f.read(4096), b"")` — 파일을 4096바이트씩 끊어 읽는 관용구. 두 번째 인자 `b""`가 **센티널**로, `read`가 빈 바이트를 반환(=EOF)하면 반복 종료. 대용량 파일도 메모리에 통째로 안 올리고 스트리밍으로 해시한다.
- `.update(chunk)` — 청크를 누적. 해시는 스트리밍 가능해서 조각조각 넣어도 전체를 한 번에 넣은 것과 결과가 같다.
- `.hexdigest()` — 최종 해시를 16진수 문자열로.

## 실습 의도 — hello.exe vs erase.exe

두 파일의 해시를 비교하는 게 핵심이다.

- **다른 파일이면 해시가 완전히 다르다**: 내용이 1바이트만 달라도 해시 전체가 바뀐다(눈사태 효과, avalanche effect). `hello.exe`와 `erase.exe`는 다른 프로그램이므로 MD5·SHA3 모두 완전히 다른 값이 나온다.
- **같은 파일이면 항상 같은 해시**: 파일을 복사하거나 이름만 바꿔도 내용이 같으면 해시는 동일. 그래서 악성코드 **동일성 판별**에 쓴다.

## MD5 vs SHA3 — 왜 둘 다?

| | MD5 | SHA3-512 |
|---|---|---|
| 출력 길이 | 128비트(32 hex) | 512비트(128 hex) |
| 충돌 저항성 | **깨짐** (의도적 충돌 생성 가능) | 안전 |
| 속도 | 빠름 | 느림 |
| 용도 | 빠른 파일 식별·인덱싱 | 무결성 보증이 중요한 경우 |

MD5는 충돌 공격이 실증되어 **보안 목적으론 부적합**하지만, 악성코드 DB의 빠른 조회 키로는 여전히 널리 쓰인다(VirusTotal 등이 MD5/SHA1/SHA256 병기). 실습에서 둘을 같이 뽑아보는 이유는 **속도와 안전성의 트레이드오프**를 체감하기 위함이다.

## 악성코드 분석에서의 활용

1. **샘플 식별**: 수집한 검체의 해시를 VirusTotal·MalwareBazaar에 조회 → 알려진 악성코드인지 즉시 확인.
2. **변조 탐지**: 정상 시스템 파일의 해시를 미리 저장해 두고(화이트리스트), 나중 값과 비교해 감염·패치 여부 판단.
3. **IOC(침해지표)**: 사고 대응 보고서에 검체 해시를 기재해 다른 조직이 같은 위협을 탐지하도록 공유.
4. **패킹 확인**: 원본과 UPX 패킹본은 해시가 다르다 → 같은 악성코드의 변종(패킹 변형)을 해시만으로는 못 묶는 한계도 함께 이해.

## 실행

```bash
cd hash/md5
python hash.py
```

`hello.exe`, `erase.exe`가 같은 폴더에 있어야 한다. `sha/` 폴더에는 SHA 실습용 파일(WinRAR 배포본 등)이 들어 있어, 큰 파일에서의 해시 계산·비교를 연습할 수 있다.
