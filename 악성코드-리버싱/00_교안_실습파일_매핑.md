- **디버거**: x32dbg (교안은 OllyDbg 1.10 기준, Part 2에 `odbg110/` 동봉). 조작은 거의 동일.

- **언패킹**: `upx.exe -d`
- 샘플 압축 암호: 실습자료 `rev123`, 실제 악성코드 샘플 `infectedds` (VM 격리 환경에서만)

## 디버거 기초 (Hello World.exe)

교안 p19~27의 핵심 조작:

- **F2** — Break Point 설정/해제
- **F7 / F8** — Step Into / Step Over
- **F9** — 실행(Run)
- **Ctrl+G** — 주소로 이동(덤프/디스어셈블리)
- **Alt+M** — Memory Map (섹션·권한 확인)
- **스페이스** — 명령어 어셈블(패치)
- **우클릭 → Search for → All referenced strings / Intermodular calls** — 문자열·API 호출 추적

## PE 포맷 (hello pe.exe)

교안 p60~88에서 다루는 구조체 계층. 리버싱에서 가장 자주 보는 필드만 요약:

| 구조체 | 핵심 필드 | 의미 |
|---|---|---|
| `IMAGE_DOS_HEADER` | `e_magic`(MZ), `e_lfanew` | NT 헤더 위치 |
| `IMAGE_NT_HEADERS` | `Signature`(PE\0\0) | PE 시작 |
| `IMAGE_FILE_HEADER` | `Machine`, `NumberOfSections` | CPU·섹션 수 |
| `IMAGE_OPTIONAL_HEADER` | `AddressOfEntryPoint`, `ImageBase`, `SizeOfImage` | EP·로드 주소 |
| `IMAGE_DATA_DIRECTORY` | Export/Import/IAT 등 | 디렉터리 위치 |
| `IMAGE_SECTION_HEADER` | `Name`, `Characteristics` | 섹션 권한(r/w/x) |

- **RVA → VA**: `VA = ImageBase + RVA`
- **IDT(Import Directory Table)**: 어떤 DLL의 어떤 함수를 쓰는지 목록
- **IAT(Import Address Table)**: 로더가 실제 함수 주소를 채워 넣는 표. `call dword ptr [IAT]` 형태로 호출됨

## 패킹 / 언패킹 (UPX)

- 패킹: `upx.exe -o <출력> <대상>` → 섹션이 `UPX0/UPX1`로 바뀌고 EP가 언패킹 스텁을 가리킴
- 언패킹 원리: 스텁 실행 → 원본 코드 메모리 복원 → **OEP(Original Entry Point)** 로 점프
- 수동 언패킹: 스텁의 마지막 `jmp OEP`를 찾아 그 지점에서 덤프 + IAT 복구
